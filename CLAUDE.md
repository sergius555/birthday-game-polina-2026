# Проект: Космический квест на день рождения (бортовой компьютер на iPad)

## Что это
Квест на день рождения дочки Полины (9 лет). Тема — «Тайна третьей планеты» (Кир Булычёв).
Полине выдают iPad как «бортовой компьютер» корабля. Это набор HTML-страниц, которые
открываются во вкладках. Всё на русском (современная орфография). Открывать на iPad,
лучше через хостинг (Netlify Drop) — из-за iframe локально (file://) вкладки не работают.

Стиль: тёмный космос, звёзды + падающие звёзды. Шрифты: Playfair Display (заголовки),
Oswald (метки/HUD), PT Serif (текст), Share Tech Mono (техно-подписи).
Цвета: фон #05060e, циан #5fd0e6, акцент-янтарь #e6b552, текст #e7e3d5.

## Главная идея (связь подарка и квеста) — без спойлеров
Крыс украл **посылку** (это подарок; НАЗЫВАТЬ ТОЛЬКО «посылка», НЕ «рюкзак», содержимое
не раскрывать — сюрприз). Полина проходит задания, идёт по следу Крыса от планеты к
планете, в финале возвращает посылку и открывает её (задумка: кодовый замок, цифры кода
собираются по заданиям).

## Персистентность — единое состояние (НЕ куки)
Куки не сохранялись на `file://` (письмо Алисы всплывало каждый раз) → перешли на общее
хранилище. Источник правды — ВЕРХНЕЕ окно `index.html`: `localStorage['edinorog_state']`
(переживает закрытие вкладки) + дубль в `window.name` (переживает F5 даже там, где Safari
блокирует storage на file://). Вкладки-iframe сами ничего не хранят — спрашивают/обновляют
состояние у оболочки через postMessage:
  - `req:state` → оболочка отвечает `state:<json>` (и сама шлёт его на `load` каждого iframe);
  - `set:<key>:<val>` → оболочка обновляет и сохраняет.
Объект состояния: `{ level, callSeen, mailRead:[ids], mailLater }`.
  - `callSeen` (или в `mailRead` есть 'alisa') → вызов Алисы больше не показывается;
  - `mailRead` — прочитанные письма; `mailLater` — что «Зелёный» доставлен (после F5 остаётся);
  - `level`: 1 = нужно ввести планету; 2 = «Венера» введена → runner; 3 = runner пройден →
    сразу экран прибытия (раннер повторно не запускается). Дальше уровни — тем же `persistLevel(n)`.
Каждый пройденный шаг не показывается заново.
